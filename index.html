Comments on the use of C2Perform

Upon closely analyzing the C2Perform system, we recognize that as an engagement team we are not fully leveraging the tool, mainly due to repetitive tasks based on other programs or implementations we already have. Below is a list of system benefits, indicating whether we use them or not, along with related comments:

Features C2Perform Comments
Quality Y Used for quality audit tracking and reporting results.
Learning Management System N Not used due to internal learning processes using DTPs and other approved documents.
Task Management N No need to assign tasks due to manual tracking and repetitive work to add them to the tool.
Coaching Y Used to provide feedback based on team results. The team uses the application once a month to close audits.
Communication Center N No need to use this feature as the team has a communications committee. (Facebook-type communication on the tool's screen).
Knowledge Management N Repetitive work to add information to the tool.
Rewards and Recognition N Not active and replaced by disruptive rewards including more holistic rating.
Gamification N No need to use badges or more. Not implemented for any use in Engagement.
Attendance N Repetitive work to add information to the tool.

The only benefit I could identify is having a system with its own support system (loaded through assignment), we just log in to the system and enter the required information to audit a customer interaction without more work. That said, I think we can replace the system with a well-created Microsoft form. The Microsoft form can present the same features which are:

  • Audit tracking
  • Rating
  • ID of the interaction being audited
  • Auditor
  • Type of interaction
  • Different scorecards
  • Scorecard type depending on the interaction and BU
  • Availability of scorecard ratings and points
  • Reports
  • Quality audit results

In conclusion, I would say we do not exploit the tool as much as we could, but it is also not necessary, we are covered, and I would see it as repetitive work to do so. I would recommend that we implement a Microsoft form to replace the system.

Comments on our Q2A application

Our Q2A application, developed as a customized PowerApp, is designed to provide a highly adaptable and efficient quality evaluation experience. Below are some of the key features and benefits of our Q2A application:

  • Call Rating: Ratings are done through a customized form, allowing complete adaptation to the specific needs of each team.
  • Audit Tracking: Custom implementation allows detailed tracking of audits performed.
  • Reports: Integrable with Power BI, offering advanced analysis tools developed according to user needs.
  • Analysis: Basic analysis is available initially. Advanced charts and graphics can be added in a future admin version using Power BI based on the data collected in the ratings.
  • User Interface: Although it requires an initial design, the interface is highly customizable to meet user preferences.
  • Integration with Other Systems: High integration with the Microsoft ecosystem, facilitating communication and workflow within the organization.
  • Customization: Fully adaptable to the specific needs of the business, providing flexibility and scalability.
  • Data Security: Full control over data security, with integration of Microsoft policies to ensure regulatory compliance.

In summary, our Q2A application not only replaces the functions of C2Perform but also provides a more flexible and scalable platform tailored to the specific needs of our organization. Implementing a Microsoft form allows for more efficient and personalized management of quality audits, reducing dependency on external systems and optimizing internal workflow.
